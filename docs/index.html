Event Based Project (EBP)

Multi-Platform Embedded Framework

EBP is a high-performance, event-driven embedded framework designed with a strict Hardware Abstraction Layer (HAL). It enables the same application logic to run seamlessly across Linux systems and bare-metal microcontrollers.

Software Architecture

The core philosophy of EBP is the strict decoupling of Event Producers (hardware, interrupts, drivers) and Event Consumers (business logic).

Event Loop Lifecycle

  • Initialization: Peripheral validation and system startup checks
  • Event List: Centralized queue collecting system events
  • Dispatcher: Pops and processes events sequentially
  • Recovery: Automatic restart on critical failures

Extensibility Model

Adding a Producer

New hardware inputs or sensors are added by generating an event via EventEkle(EVENT_TYPE). No business logic changes are required.

Adding a Consumer

Business logic is implemented in centralized event handlers. New functionality is added by extending the event processing switch.

Cross-Platform HAL

EBP separates hardware-dependent code behind a unified HAL layer, allowing the same application logic to execute across multiple platforms.

Build & Usage

The project includes dedicated build configurations for both embedded and desktop environments.

  • Linux: gcc, cmake, make
  • STM32: arm-none-eabi-gcc, STM32CubeIDE, Keil
  • Projects are built directly via Eclipse / CubeIDE
